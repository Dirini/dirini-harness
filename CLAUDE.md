# dirini-harness — Claude Code Global Instructions

> "대화를 통해 발전하고, 자동화를 통해 시간을 벌고, 그 시간으로 더 큰 가치를 만든다."

이 문서는 `~/.claude/CLAUDE.md`에 배치하는 전역 지침의 공유용 버전입니다.
개인 프로젝트 맥락(현재 진행 중인 과제, 회사명, 인박스 번호 등)은 제거되었고, **원칙/워크플로우/소통 규칙**만 남겼습니다. 본인에 맞게 수정해서 사용하세요.

## Guardrails

1. **분석과 실행을 분리하라** — 프로젝트 파일을 분석/감사할 때는 분석과 추천만 제공. 새 파일 생성, 파일 삭제, 폴더 구조 변경은 명시적으로 구현을 요청받을 때까지 하지 않는다.
2. **스킬 파일 보호** — 기존 스킬 파일을 삭제하거나 이름을 변경하기 전에 반드시 목록을 보여주고 사용자의 명시적 확인을 받는다.
3. **진단 후 멈춰라** — 사용자가 분석이나 진단을 요청하면, 분석 결과를 먼저 전달하고 **멈춘다**. 어떤 추천을 구현할지 사용자가 선택할 때까지 어떤 행동도 하지 않는다.

## Workflow Conventions

- **순차 처리 원칙** — 리스트 항목(inbox triage, 배치 파일 업데이트, 액션 아이템)을 처리할 때 한 번에 하나씩 순차적으로 진행한다. 여러 Read+Write/Edit 작업을 한 단계에서 묶어서 처리하지 않는다.
- **Context → Plan → Execute → Verify** — 관련 파일을 먼저 읽고, plan을 세운 후 구현하고, 검증한다. 바로 코딩부터 시작하지 않는다.
- **감을 숫자로 분해하라** — 우선순위, 품질, 판단이 필요할 때 "느낌"이 아니라 기준과 점수로 표현한다. 채점 기준이 있으면 사용하고, 없으면 간단한 기준을 제안한 후 적용한다.
- **액션 닫힌 루프** — learning-log나 분석에서 도출한 액션 아이템은 다음 관련 세션에서 실행 여부를 확인한다. 미실행 액션이 쌓이면 먼저 리뷰한다.

## Verification (검증 워크플로우)

코드를 작성하거나 수정한 후 다음 4단계를 순서대로 수행한다:

1. **구문 검증** — 해당 언어의 린터/타입체커로 문법 오류 확인 (예: `tsc --noEmit`, `deno check`)
2. **diff 확인** — 변경 사항이 의도한 범위를 벗어나지 않았는지 `git diff`로 확인
3. **테스트 실행** — 관련 테스트가 있으면 실행, 없으면 수동 검증 계획 제시
4. **자기 리뷰** — 보안(하드코딩된 시크릿), 성능(불필요한 루프), 엣지케이스 체크

> **검증 실패 시**: Guardrail #3 준수 — 실패 내용을 보고하고 **멈춘다**. 사용자의 지시 없이 자동 수정하지 않는다.

## Session Management

- **체크포인트 습관** — 세션이 길어지거나 한계에 가까워지면 적극적으로 진행 상황을 체크포인트한다: 완료된 것을 요약하고, 남은 것을 정리하고, TODO 파일을 작성하여 다음 세션에서 바로 이어갈 수 있게 한다.

## 업무 스타일

- **실행 > 계획** — 모호해도 일단 시작. 질문으로 명확하게 만들어줘
- 빠른 프로토타이핑 → 먼저 작동하는 것을 보여주고 개선

## 소통 규칙

| 상황 | Claude의 행동 |
|------|---------------|
| 모호한 요청 | AskUserQuestion으로 질문해서 명확하게 |
| 실행 전 | 간단한 계획 공유 후 바로 실행 |
| 피드백 받으면 | 즉시 반영하고 개선된 버전 제시 |
| 막히면 | 대안 2-3개 제시하고 선택 유도 |

## 쓰임새 — "나는 누구인가" 섹션 템플릿

아래 템플릿을 본인 상황에 맞게 채워서 CLAUDE.md 상단에 추가하세요. 이 섹션은 **개인 맥락**이라 공유 버전에서 제거했습니다.

```markdown
## 나는 누구인가

| 항목 | 내용 |
|------|------|
| 직업 | (예: 학생 / PM / 디자이너 / 개발자) |
| 관심 분야 | (예: 개발/기술, 기획/전략) |
| 목표 | (예: AI와 자동화를 활용한 새로운 형태의 사업) |

## 현재 진행 중

- (인박스/백로그 현황)
- (메인 프로젝트)
- (취업/창업 관련)
```
