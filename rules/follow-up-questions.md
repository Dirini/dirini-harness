# 꼬리질문 원칙 (Follow-up Questions Rule)

## 인터뷰 모드 트리거

다음 신호가 감지되면 **interview 스킬**을 활성화하여 본격 인터뷰 진행:
- `/interview`
- "깊게", "깊이", "구체화", "제대로", "본격적으로"
- "인터뷰 해줘", "질문 해줘"

→ 5-7라운드 깊이 있는 질문, 완료 후 스펙 파일 생성

---

## 모호함 판단 → Clarify 3종 모드 자동 선택

요청을 받았을 때 다음 질문을 스스로에게 던져라:
> "이 요청을 2가지 이상 다른 방향으로 해석할 수 있는가?"

### 모호함 신호 → 모드 매핑

| 신호 | Clarify 모드 | 행동 |
|------|-------------|------|
| "뭘 해야 할지 모르겠어", 우선순위 불명확 | **Unknown** | Known/Unknown 매트릭스로 분석 |
| 콘텐츠 작업 + 형식/플랫폼 불명확 | **Metamedium** | 콘텐츠(무엇?) vs 형식(어떻게?) 분리 질문 |
| "좀", "적당히", "알아서", 주관적 표현 | **Vague** | 가설 2-3개 + 구체적 결과물 미리보기 |
| 구체적 파일/기능 있고 해석 1개 | - | 바로 실행 |

> 상세 워크플로우: `.agent/workflows/clarify.md`

## 행동 지침

### 명확한 요청 (해석 1개)
→ 바로 실행

### 모호한 요청 (해석 2개 이상)
→ 위 표에서 모드 선택 → 꼬리질문 1-2개 던지기

## 꼬리질문 형식

1. **객관식 선택지** 제시 (2-3개)
2. **간결하게** (장황한 설명 X)
3. **최대 2개** 질문
4. `AskUserQuestion` 활용 권장

### 예시

❌ 나쁜 예:
```
"정리해달라고 하셨는데, 정리의 범위와 방식에 대해
여러 가지 해석이 가능합니다..."
```

✅ 좋은 예 (Vague 모드):
```
어떤 방식으로 정리할까요?
1. 코드 리팩토링 → 함수 분리, 네이밍 개선
2. 파일 구조 변경 → 폴더 재배치
3. 문서화 → README/주석 추가
```

## 예외: 꼬리질문 생략

- 사용자가 "일단 해봐", "알아서 해" 명시
- 이전 대화에서 맥락이 충분히 확보됨
- 되돌리기 쉬운 작은 변경
