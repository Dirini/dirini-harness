# Cover Letter Skill (자소서 작성)

자소서를 작성하고, 7개 항목 100점 만점으로 채점하고, 글자수 범위에 맞춰 반복 개선합니다.

## Triggers

- `/cover-letter`, `/자소서`
- "자소서 써줘", "자소서 도와줘", "지원서 작성"
- "[회사명] 자소서", "[회사명] 지원"

## Phase 1 — Context Load

1. 확인할 정보:
   - 회사명, 직군, 글자수 제한 (기본 1000자)
   - 자소서 문항 텍스트 (사용자가 붙여넣기)
   - 포트폴리오/보고서 PDF 경로 (증거 추출용)

2. 기존 자료 확인:
   - `~/Documents/New project/docs/` 아래 해당 회사 폴더 존재 여부
   - `master-experience-sheet.md` 경험 목록
   - 이전 세션에서 작성한 자소서가 있으면 로드

3. 증거 추출:
   - PDF를 읽고 **정량 증거**를 티어별로 분류:
     - Tier 1: 외부 검증 (학술지 게재, 수상, 언론 보도)
     - Tier 2: 행동 변화 지표 (재구매율, NPS, 재방문율)
     - Tier 3: 규모/도달 수치 (참여자 수, 조회수, 모금액)
     - Tier 4: 재무/프로세스 (순이익, 비용 절감) — 소규모일 경우 약한 증거

## Phase 2 — Company Research

> `/company-research` 커맨드를 활용하거나, 아래 체크리스트 수동 수행

회사 연결성 점수를 높이기 위한 필수 리서치:
- [ ] 회사 고유 숫자 1개 (매출, 시장점유율, 해외비중 등)
- [ ] 해당 직군 관련 제품/브랜드명 1-2개 (플래그십 외)
- [ ] 내부 철학/문화 키워드 1개 (PM End-to-End, OKR 등)
- [ ] 최근 1년 이니셔티브/캠페인 1개

**금지**: JD 문구를 그대로 복사해서 회사연결성으로 쓰지 않는다.

## Phase 3 — Draft

문항별로 초안 작성:

### 구조 가이드

**지원동기+강점 유형 (Q1 패턴)**:
```
[회사의 구체적 방향 + 내가 선택한 이유]
→ [내 경험이 그 방향과 교차하는 지점]
→ [강점의 정량 증거 (Tier 1-2 우선)]
→ [입사 후 기여 방향 1문장]
```

**문제의식+개선 유형 (Q2 패턴)**:
```
[현장에서 포착한 구체적 문제]
→ [기존 방식의 한계 진단]
→ [내가 선택한 접근법 + 실행 내용]
→ [결과 수치 (Tier 3)]
→ [교훈 + 직무 연결 1문장]
```

### 작성 원칙

1. **모든 문장이 새로운 정보**를 추가해야 한다 (반복 문장 = 글자수 낭비)
2. **Tier 1 증거는 비협상** — 학술지 게재, 수상 등은 반드시 포함
3. **회사 연결은 고유 팩트로** — JD 복사 금지
4. 글자수 타겟: `[제한 - 50]자 ~ [제한]자`

## Phase 4 — Score (7개 항목, 100점)

| 항목 | 배점 | 기준 |
|------|------|------|
| 문항적합성 | 25 | 질문이 묻는 것에 정확히 답했는가 |
| 직무적합성 | 20 | 경험이 해당 직무 역량과 직결되는가 |
| 본인기여도 | 15 | "직접", "주도", "설계" 등 본인 행동이 명확한가 |
| 문제-행동-결과 | 15 | PAR/STAR 구조가 완결되었는가 |
| 정량근거 | 10 | 구체적 숫자가 주장을 뒷받침하는가 |
| 회사연결성 | 10 | 이 회사여야 하는 이유가 고유 팩트로 드러나는가 |
| 문장밀도 | 5 | 반복/빈 문장 없이 밀도 있게 썼는가 |

### 채점 방법

각 항목별 점수 + 1줄 근거를 테이블로 출력.
총점 90점 미만이면 가장 낮은 항목부터 개선 제안.

## Phase 5 — Character Count + Iterate

```bash
echo -n "[draft]" | wc -m
```

- 범위 밖이면 밀도를 조절 (내용이 아닌 표현 수준에서)
- 반복 문장 제거 → 확보된 글자수로 탈락 증거 복원
- **Guardrail**: 수정 1회 후 사용자에게 확인받고 다음 반복 진행

## Phase 6 — Final Output

1. 최종 텍스트
2. 최종 스코어카드 (7항목 테이블)
3. 글자수 (공백 포함)
4. 저장 제안: `~/Documents/New project/docs/[company]/cover-letter-[date].md`

## Codex/AI 초안 검수 체크리스트

사용자가 AI 생성 초안을 가져왔을 때 반드시 확인:
- [ ] Tier 1 증거(학술지, 수상)가 탈락되지 않았는가
- [ ] 약한 숫자(소규모 순이익 등)가 강한 숫자(행동변화 지표)를 밀어내지 않았는가
- [ ] 회사연결 부분이 JD 복사가 아닌 고유 팩트인가
- [ ] 마지막 2-3문장이 같은 말을 반복하지 않는가
- [ ] 글자수 범위 안에 있는가
